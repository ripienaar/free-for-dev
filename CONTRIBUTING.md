# Contributing to free-for.dev

Thank you for your interest in contributing — this project is community-maintained and contributions are welcome.

Scope
- This list contains "as-a-Service" offerings with free developer tiers. Self-hosted projects are out-of-scope.

How to contribute
- Fork the repository and create a feature branch.
- Make small, focused changes (one logical change per PR).
- Update or add entries in `README.md` (or the relevant docs page) following existing formatting.
- Open a Pull Request describing your change clearly and reference any related issue.

Formatting rules
- Keep entries short: Name — short description — URL.
- Use Markdown only. Keep line length reasonable (wrap at ~100 chars).
- When adding an entry, include the official URL and a one-line description.

PR process
- Ensure your PR has a clear title and description.
- Add a checklist in the PR (use the provided template).
- CI will run markdown linting and a link-checker (if configured). Fix issues before merging.

Code of Conduct
- This project follows the [Code of Conduct](CODE_OF_CONDUCT.md). By participating you agree to its terms.

Thank you — maintainers
